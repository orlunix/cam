External relay

              

                Connect Desktop to an existing CAM Hub through a relay
                endpoint. The relay forwards REST/WS to a Hub you cannot
                reach directly. The CAM API token is profile-managed on
                the source side and injected by the relay; only the Relay
                URL and Relay token are needed here.

Add host manually

              

                Register a remote node by SSH. The embedded Hub stores
                host/user/port/path/env locally; the private key file
                is referenced by path only — its contents are never
                read or copied (CAM-DESK-DIRECT-018).
              

              
              
              
              
              
              

              

                
                
                
              


              

                
                
              


              

                

                  Use the running SSH agent or the default key search
                  (~/.ssh/id_*). No key file or password
                  is stored by Desktop in this mode.
                

              


              
              

                Editing host fields applies to all
                context(s) on this host. Workspace fields (name,
                remote path, env setup) are managed per context and are
                not changed by this form.
              

              

                Editing context "…".
                Context name is read-only. Remote path and env setup are
                changed here. Host fields are shown locked/grey because
                contexts are bound to their host; use
                Edit Host to change host settings.
              

              

                Adding a new context under host
                "…".
                Host fields (host, user, port, auth, key file) are
                inherited from the existing host. For password-auth
                hosts you must enter the password again — the embedded
                Hub does not auto-clone credential references to new
                contexts.
